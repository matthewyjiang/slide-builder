# Deck design packages

When a design package is active, `DESIGN.md` is the primary visual contract. Follow it before generic preferences. The system prompt includes its full text and lists supplementary package files that can be read on demand.

Read `TEMPLATE-REFERENCE.md` when it is listed and you need template-specific layout names, placeholders, examples, or constraints. Do not load every supplementary file without a reason. When package templates exist, the user must choose explicitly between a blank deck and each template. Never silently select `template.pptx`.

## Design method

- Start with the audience, purpose, and one-sentence takeaway.
- Give each slide one job and one dominant visual idea.
- Establish repeatable margins, columns, type scales, colors, and spacing.
- Prefer concise headlines and visual evidence over paragraphs.
- Use charts for quantitative comparison, diagrams for systems or sequence, and images for concrete subjects or emotional emphasis.
- Preserve hierarchy and legibility at presentation distance.
- Vary composition enough to maintain rhythm while retaining the package's visual language.
- Validate and render all slides, not only the active slide.

When no package is active, use restrained defaults: a high-contrast neutral base, one purposeful accent color, a limited typography scale, generous whitespace, and consistent alignment. Do not invent brand claims, logos, or proprietary styling.
